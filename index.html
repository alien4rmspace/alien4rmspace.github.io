Project 01

CUDA-Accelerated Mandelbrot Renderer

Parallel fractal rendering in C++ and CUDA, profiled with NVIDIA tooling to validate a real speedup over a CPU baseline.

GitHub

Built a Mandelbrot renderer in C++ with CUDA acceleration, distributing per-pixel fractal work across the GPU to increase rendering throughput substantially compared to a CPU implementation.

Instrumented CPU and GPU execution with NVIDIA Nsight Systems and NVTX markers to separate phases of execution, inspect bottlenecks, and make performance claims based on traces rather than guesswork.

The result was about a 370x speedup, with example frame timings dropping from about 374 ms on CPU to about 1.0 ms on GPU. The project emphasizes parallel work distribution, throughput, deep-zoom rendering, and performance-oriented design backed by measurement.

  • Mapped pixels in parallel on the GPU for fractal generation
  • Used NVTX markers to make traces easier to interpret
  • Focused on measurable speedup instead of vague optimization claims

Project 02

GPU-Accelerated Pokemon Card Classifier

End-to-end image classification pipeline tuned for training and inference throughput, not just model accuracy.

GitHub

Built a Pokemon card recognition pipeline in Python using PyTorch and CUDA, fine-tuning a ResNet-18 model on a custom dataset with automated data collection and preprocessing.

Profiled the training pipeline with NVIDIA Nsight Systems and NVTX to understand where time was going across model execution and data loading, then tuned the input pipeline and DataLoader settings to reduce bottlenecks.

Reduced 5-epoch training time from about 200 seconds to about 66 seconds and reduced inference time from about 15 seconds to about 8 seconds. The project shows machine learning systems thinking, GPU-aware profiling, and a focus on end-to-end throughput.

  • Fine-tuned ResNet-18 on a custom image classification dataset
  • Explored DataLoader and num_workers tradeoffs for throughput
  • Treated the ML pipeline as a performance system, not just a model

Project 03

Multiplayer Zombie RTS / Survival Game

Unity multiplayer project centered on server-authoritative logic, synchronized gameplay systems, and architecture that can scale beyond a local prototype.

GitHub

Built a multiplayer zombie survival and RTS-style game in Unity using Netcode for GameObjects, with a focus on server-authoritative state, networked interactions, and the complexity of syncing units, zombies, and player actions.
